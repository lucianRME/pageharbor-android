# Branding

PageHarbor's brand should support a focused, privacy-first document scanning tool. This guide describes direction, not final visual assets.

## Brand Purpose

PageHarbor exists because scanning documents should not require:

- Accounts.
- Subscriptions.
- Advertising.
- Tracking.
- Proprietary cloud storage.

The product should feel calm, trustworthy, and respectful of user privacy.

Never use fear-based marketing. Never criticize competitors. Focus on positive messaging about user control, clear choices, and local-first document handling.

## Brand Promise

Scan documents without giving them away.

This is the primary brand promise. It should guide product copy, onboarding, store listings, screenshots, and public documentation. It must be supported by implementation and validation before stronger privacy claims are made.

## Core Values

- Privacy: Document handling should minimize data exposure and avoid tracking, advertising, accounts, and proprietary cloud storage.
- Simplicity: The product should make scanning, exporting, saving, and sharing easy to understand.
- Transparency: Explain what PageHarbor does, what it does not do, and what external platform components may handle.
- Reliability: Scanning and export flows should be predictable, recoverable, and respectful of cancellation or failure.
- User control: Users choose when to scan, where to save, and whether to share.

## Product Personality

PageHarbor should feel:

- Calm.
- Professional.
- Modern.
- Honest.
- Lightweight.
- Respectful.

PageHarbor should never feel:

- Aggressive.
- Noisy.
- Sales-driven.
- Flashy.
- Complicated.

## Voice And Tone

Writing should be:

- Short.
- Clear.
- Human.
- Confident.
- Technically accurate.

Avoid:

- Buzzwords.
- Exaggerated claims.
- Marketing cliches.
- "AI-powered".
- "Revolutionary".
- "Next-generation".
- "Military-grade".
- "100% secure".

Never claim certified privacy, perfect security, guaranteed offline behavior, or absolute anonymity unless those claims are verified and documented.

## Visual Identity

Intended visual direction:

- Generous white space.
- Rounded Material 3 components.
- Minimal iconography.
- Soft blue accents.
- Clean typography.
- Uncluttered layouts.

Avoid:

- Gradients.
- Skeuomorphism.
- Heavy shadows.
- Unnecessary illustrations.
- Decorative graphics.

## Selected Mark Concept

The selected PageHarbor mark is a simple upright document with a folded top-right corner resting in a soft curved harbor shape.

The curve should suggest safety, support, and arrival. It must not read as a boat, anchor, shield, lock, scanner frame, QR code, or wave illustration.

The mark should stay minimal, recognizable at Android launcher sizes, and suitable for adaptive icon masking.

## Color Direction

Initial approved color values:

- Primary blue: `#245FC7`.
- Primary dark: `#173E85`.
- Accent blue: `#8CB9F1`.
- Dark slate: `#172033`.
- Light surface: `#F8FAFD`.
- White: `#FFFFFF`.
- Error: `#B3261E`.

Dark theme support:

- Dark background: `#101827`.
- Dark surface: `#172033`.
- Dark text: `#E6ECF5`.
- Dark error: `#F2B8B5`.

Accessibility takes precedence over strict color matching. If a color pairing does not meet WCAG AA contrast where applicable, adjust the pairing before preserving the exact brand color.

## Logo Direction

The logo should communicate:

- Trust.
- Documents.
- Safety.

Avoid:

- Cameras.
- Scanners.
- PDF icons.
- Locks.
- Shields.
- Padlocks.
- Anchors.

The logo should feel timeless.

Possible concepts:

- A simple page form with a subtle harbor or safe-arrival reference.
- A folded-corner document shape with a calm horizontal baseline.
- A minimal document outline paired with a small negative-space inlet.
- A refined wordmark using clean, readable type with no decorative effects.

## App Icon Direction

The app icon should remain recognizable at 48x48 pixels.

- Keep it extremely simple.
- Use one primary shape.
- Use minimal detail.
- Do not place text inside the icon.
- Use an adaptive icon foreground vector.
- Use a calm off-white background.
- Provide a monochrome resource for themed icons.
- Keep artwork inside common adaptive icon safe areas.

## Splash Direction

Use the Android system splash screen.

- Use the PageHarbor app icon.
- Use a calm light background in light mode.
- Use an appropriate dark background in dark mode.
- Do not add artificial delay.
- Do not include "by SynapseWorks" or other text inside the splash.
- Transition directly to the Home screen.

## Terminology

Preferred:

- Scan document.
- Export PDF.
- Share.
- Privacy.
- Local processing.

Avoid unless those features actually exist:

- Upload.
- Cloud backup.
- Sync.
- Vault.
- Encrypted military-grade storage.

## Screenshots

Screenshots should be:

- Realistic.
- Clean.
- Uncluttered.
- Free of fake notifications.
- Free of fake data.
- Free of exaggerated marketing text.

## Initial Visual Assets

The repository includes initial editable SVG assets for:

- Primary brand mark.
- Horizontal logo lockup.
- Monochrome mark.
- GitHub social preview concept.
- Play Store feature graphic concept.

The GitHub and Play Store graphics are initial visual assets, not final production exports. Dimensions, text rendering, and store-specific requirements must be verified before public use.

## Accessibility

Branding must remain accessible.

- Avoid color-only communication.
- Support light and dark mode equally.
- Keep contrast, text size, and touch targets practical.

## Future Branding

If OCR or AI features are added later, they should extend the existing privacy-first brand rather than becoming the center of the product.

Privacy remains the primary differentiator.

## Non-Goals

PageHarbor is not trying to become:

- Dropbox.
- Google Drive.
- Evernote.
- Notion.
- Microsoft Lens clone.
- CamScanner clone.

It is a focused document scanning tool.
